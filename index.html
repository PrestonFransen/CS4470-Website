HW0 - Cost-Optimized Parallel Code Breaker (COPACOBANA)

Problem.

Cracking small key size ciphers. Many cryptographic algorithms require multiple independent operations. Because of this, they pair well with hardware implementations of parallelization.


COPACOBANA is a machine designed for this task utilizing commercially available field-programmable gate arrays (FPGA). Each COPACOBANA contains 20 modules with each module containing 6 FPGAs --> total 120 compute FPGAs. The data bus of each module is interfaced to the global data bus on a backplane. FPGAs on the same module can communicate via a local 64-bit data bus. Control signals are run over a separate 16-bit address bus. Memory is local to each chip.

Performance.

COPACOBANA is able to perform cryptographic operations in parallel for applications with high computational but low memory and communication requirements. DES can be broken within less than a week at an average throughput of 65.3 billion searched keys per second.

Scalability.

Larger keys that rely on DES can continue to be exploited by highly compute bound brute-force decryption. Adding more boards to the COPACOBANA and segmenting smaller portions of the key space results in a linear speedup.
